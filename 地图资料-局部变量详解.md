# 局部变量

在地图创作中，局部变量就是**对设计者有具体意义**的，**会因触发**（和动作脚本）**做出改变的**，**临时在某一局游戏起作用**的数值。  
我不知道日牛（或者说 Hares）对此有没有扩展（毕竟我很鸽），至少原版 YR 是有使用限制的：
- 它得是`bool`类型；
- 至多只能设 100 个。

> [!note]
> `bool`讲大白话就是开关。`0`表示否定、假、闭合、不是，`1`代表肯定、真、开启、是。  
> 圈里有那么些人不喜欢这种程序化表达，那我也点到为止。想了解的自己翻《计算机组成原理》。

## 变量的声明和使用

在 FA2 的编辑菜单→局部变量里，你可以点开下拉框，选中`No Name`，随便起一个名字，这样就**声明**好一个变量了。  
不出意外的话，可以看见这个变量的**初始值**是 0。当然你也可以改成 1。往高了填当然也可以，但至少在原版中毫无意义。  
除了拿`No Name`设一个新的之外，你也可以选中已有的名字，指定它们的初始值。

前面我们声明了局部变量，这里就把它用起来。触发里有一组事件和一组行为分别读写局部变量（设待操作的局部变量为x）：
- 事件 36：（指定）局部变量被设定（值为 1），即`当 x = 1 时`；
- 行为 56：设置（指定）局部变量（值为 1），即`令 x = 1`。
* 事件 37：（指定）局部变量被清除（0），即`当 x = 0 时`；
* 行为 57：清除（指定）局部变量（值），即`令 x = 0`。

> [!note]
> 那么我还是要用程序视角解释这些条件和结果。不然光看一排`x = 1` `x = 0`很容易感觉在念绕口令。  
> 在程序里判断一个变量等于某个值通常用`==`，所以我们容易得到`x == 1`（事件 36）和`x == 0`（事件 37）。  
> 然后“判断”这种事，数学的“命题”喜欢叫“若”，翻译成英文叫`if`，所以我们易得：
> ```c
> if (x == 0);
> if (x == 1);
> ```
> 命题有“若”了通常会有“则”，我们同样塞点东西到代码里：
> ```c
> if (x == 0) {  // 如果这变量值为 0
>     // 那么就在屏幕上打出这句话
>     logScreen("全体目光向我看齐！我就是纱布！");
>     x = 1;  // 再令 x 值为 1。
> }
> ```
> 至此你就懂变量是怎么一回事了。

## 高阶用法：局部变量实现“或” “非”
原版的触发—事件—行为系统要求一个触发的“所有条件”全部满足，方可执行一条条的行为。那假如要做这种判断呢：
- 如果你缺钱（< $100），**或者**你缺电，我就运一波坦克给你；
- 要求指定时间内摧毁目标（也就是计时器**没有**超时，并且目标被摧毁）。

……仅靠触发当然是木大（又是“且”逻辑又没有`else`设计）。那就只好配合局部变量了。

> [!note]
> 下面的用例以 C 语言伪代码形式给出。不然要完整描述这个逻辑不知道要截多少张图。

### “或”——殊途同归
首先缺钱缺电都是有现成触发的，我们可以先分别判断这两件事。然后设局部变量`resources not enough`，初值为 0，并在触发判断里做或门电路实现：
```c
bool res_not_enough = 0;
if (playerCashLessThan(100)) { res_not_enough = 1; }  // 钱不够，开关导通
if (playerLowPower()) { res_not_enough = 1; }  // 电不够，开关也导通
```
最后只要`resources not enough`开关一通（值为 1），就刷兵：
```c
// 书接上回
if ( res_not_enough == 1) {  // 假如开关导通了
    playEVA("EVA_ReinforcementsHaveArrived");  // 就语音提示有刷兵
    spawnTeam("0x01bF52"); // 并且把援兵刷出来
}
```
所谓“一真皆真”，这就是或逻辑的精髓。当然程序里真正的逻辑或`&&`有短路特性，这里也不再展开了。

### “非”——逆向思维
正常来说，触发是等计时器走完，去干嘛干嘛。那反过来，走完时间就意味着“没有超时”被打破，整个条件组肯定也就废了（任务失败）。
那么我们假设原本任务应该能够完成，所以伪代码这么写：
```c
bool mission_in_time = 1;  // 我的变量名取「及时」的语义，代表任务初始可以完成
if (timeElapsed(max)) /* 若超时 */  { mission_in_time = 0; } // 那就“不再及时”
if (
    mission_in_time == 1 // 假如按时完成任务
    && objectDestroyed(target) // 目标也成功干掉
) {
    logScreen("Yeah"); // 就打一句好耶
    endGameSuccess(); // 任务完成，通关收工
}
```
这样一来，触发通过判断计时器“时间过了”，反向推出“没有超时”这一条件不满足，从而实现逻辑的“非”运算。
